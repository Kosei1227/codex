# Checkpoint Template

Use checkpoints before and after long commands, traversal, live execution, cleanup, or major implementation phases.

```text
Checkpoint:

Current goal:

Acceptance criteria:

Completed since last checkpoint:

Files changed:

Commands run:

Failed validations:

Artifacts created or inspected:

Current hypothesis:

Proven:

Unproven:

Blocked:

Next step:

Resume command or entrypoint:

Stop condition status:
not reached | reached | approaching
```

Checkpoint rules:

- Keep it short enough to scan.
- Include exact paths and IDs.
- Do not use "latest", "previous", or "successful" without concrete identifiers.
- If the next step is risky, stop and request confirmation.
