# HTML Quality Checklist

Use this file as a final pass before delivering an HTML artifact.

## Document Quality

- Is the purpose obvious in the first screen?
- Does the top section help a busy reader decide whether to keep reading?
- Are facts, interpretation, and recommendation visually separated?
- Is the artifact readable without needing the chat transcript?

## Layout Quality

- Does the page have a clear visual hierarchy?
- Are sections visually distinct without becoming noisy?
- Are comparison views aligned enough to scan quickly?
- Is spacing consistent across cards, code, and diagrams?

## Diagram Quality

- Does each diagram answer a concrete question?
- Are arrows, labels, and groupings understandable without narration?
- Is SVG text large enough to read on a normal browser width?
- Is the diagram simpler than the prose it replaces?

## Code And Evidence Quality

- Are snippets short enough to scan?
- Does each snippet support a specific claim?
- Are file paths, IDs, commands, or labels included where precision matters?
- Are speculative claims labeled as assumptions or open questions?

## Interaction Quality

- Does each control have a visible payoff?
- Can the user understand what changed after using a control?
- Is there an export or copy path if the artifact is an editor?
- Would the page still be useful if JavaScript failed?

## Sharing Quality

- Does the file work as a standalone local HTML file?
- Does it avoid external assets that may break when shared?
- Is the page reasonably readable on narrow screens?
- Would a colleague understand the artifact without extra setup?
