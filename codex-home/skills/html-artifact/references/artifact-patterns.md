# Artifact Patterns

Use this file when the main request is clear but the HTML structure is not.

## Implementation Blueprint

Use for plans, specs, implementation proposals, and architecture walkthroughs.

Recommended sections:

1. Title and one-screen summary
2. Goal, scope, and non-goals
3. Current-state or problem framing
4. Proposed architecture or flow diagram
5. Key decisions and tradeoffs
6. File or system touchpoints
7. Validation or rollout plan
8. Risks, open questions, and next decisions

Useful visual patterns:

- SVG data-flow diagram
- Phased timeline
- Responsibility split cards
- Before and after comparison

## Report Or Explainer

Use for research summaries, incident reports, weekly updates, or concept explainers.

Recommended sections:

1. Executive summary
2. Key findings
3. Evidence or timeline
4. Supporting diagrams or metrics
5. Implications
6. Recommendations or next actions

Useful visual patterns:

- Timeline
- Scorecard cards
- Evidence callouts
- Annotated chart or process map

## PR Review Or Code Walkthrough

Use for explaining a diff, reviewing a subsystem, or onboarding someone to unfamiliar code.

Recommended sections:

1. Change summary
2. Why this matters
3. Risk hotspots
4. Annotated code snippets or diff excerpts
5. Control-flow or state-transition diagram
6. Findings, gotchas, and validation notes

Useful visual patterns:

- Inline margin annotations
- Severity callouts
- Module relationship map
- Request or event lifecycle diagram

## Side-By-Side Exploration

Use for comparing directions, layouts, product bets, or implementation options.

Recommended sections:

1. Brief framing
2. Comparison grid
3. Per-option tradeoff labels
4. Recommendation or decision lens

Useful visual patterns:

- Multi-column card grid
- Shared evaluation rubric
- Highlighted preferred option

## Prototype Or Animation Playground

Use for motion tuning, component experiments, or interaction feel.

Recommended sections:

1. Live preview
2. Adjustable controls
3. Presets or variants
4. Export or copy parameters

Useful visual patterns:

- Slider panel plus preview pane
- Variant gallery
- Token readout or timing breakdown

## Throwaway Editor

Use for ticket bucketing, prompt tuning, config editing, or one-off structured curation.

Recommended sections:

1. Main editing surface
2. Constraint warnings
3. Summary of current state
4. Export action

Useful visual patterns:

- Drag-and-drop columns
- Live preview panel
- Diff view
- Copyable prompt or JSON payload
